# code-context-builder

![Python 3.11+](https://img.shields.io/badge/Python-3.11%2B-blue.svg) ![Docker](https://img.shields.io/badge/Docker-supported-2496ED.svg)

プロジェクトのソースコードやディレクトリ構成を抽出し、LLM（ChatGPT, Claude, Geminiなど）にコンテキストとして渡すためのMarkdownファイルを自動生成するツールです。

**このツールが解決できる課題:**
* LLMに複数ファイルのコードを読ませる際、コピペの手間やファイル構造の説明の手間を削減します。
* LLMの特徴として、多くの文章(=トークン)を渡すと注意が散逸し精度が低下します。そのため、予めどのファイルを抽出するかを設定ファイルへ書き留めておくことで、ワンコマンドで作業中の最新コード状況を渡せます。
* `ast` モードを使えば、「実際にその関数から呼び出されている依存コード」だけをピンポイントで抽出し、トークン数を劇的に削減できます。

## 必須要件
* Bashが実行可能な環境 (Linux, macOS, WSL2など)
* Docker および Docker Compose

## How to use?
**1. DockerとBashが使用できる環境を構築**

**2. `configs/` にあるJSONファイルを参考に、対象ディレクトリや除外ルールを設定**
```json
{
    "target_dir": "../../",
    "includes": [
    ],  
    "excludes": [
        "output"
    ],
    "output_file_name": "sample_all_context.md",
    "enable_tree": true,
    "max_excluded_depth": 2
}
```

**3. 作ったJSONファイルと実行モードを指定して `run.sh` 実行**
```bash
# 基本構文
./run.sh <mode> <config_json_path>

# 実行例: all モード
./run.sh all ./configs/all/sample.json

# 実行例: ast モード
./run.sh ast ./configs/ast/sample.json
```

**4. ファイルが出力されます**

## 主な機能
* **2つの抽出モード**: 用途に合わせて、ディレクトリベースの全抽出（`all`）と、依存関係を追跡する解析抽出（`ast`）を選択可能です。
* **トークン数の自動計算**: OpenAIの `cl100k_base` エンコーディングを利用し、生成されるマークダウンのトークン数を事前に見積もれます。
* **LLM最適化**: コード内の完全な空行を削除してトークンを節約しつつ、ファイルサイズが大きすぎるファイル（100KB以上）やバイナリファイルは自動でスキップします。
* **自動マスキング保護**: 組み込みのシークレットスキャンエンジン（Gitleaks）により、APIキーやプライベートIPを自動検知して `[MASKED_SECRET_1]` のように連番で安全に置換します。コードの文脈を壊さずに保護します。
* **ツリー構造の生成**: 抽出対象と除外対象を整理したディレクトリツリーをMarkdownの先頭に付与します。
* **クリーンな実行環境**: Docker Composeを利用するため、ホストのPython環境を汚さずに実行可能です。

## 🛡️ セキュリティと注意事項

本ツールを利用してLLMへソースコードを送信する際は、情報漏洩を防ぐために以下の点に必ず留意してください。

### 1. 自動マスクの限界と「運用（除外）によるカバー」
本ツールは、APIキーなどの認証情報や一部の内部ネットワーク情報（プライベートIP等）を自動的にマスクしますが、**以下のような情報はツール単体で安全に検知・マスクすることができません**。

* **個人情報 (PII):** テストデータ内の顧客氏名、メールアドレス、電話番号など。
* **クラウドインフラの公開識別子:** S3バケット名、AWSアカウントID、パブリックIPなど。
* **内部事情を含むコメント:** `// FIXME: ここに脆弱性あり` などの社外秘メモ。
* **コアビジネスロジック:** 競合に知られたくない独自のアルゴリズムや計算処理。

**【対策】**
自動マスク機能に依存しすぎず、機密性が極めて高いロジックが含まれるディレクトリ（例: `core_algorithms/`）や、ダミーの個人情報が集まるテストディレクトリ（例: `tests/fixtures/`）は、**必ず設定ファイルの `excludes` に指定し、抽出対象から除外する運用を徹底してください。**(エンタープライズ保護が適用されているLLMサービスや、ローカルLLMなどに渡す場合は問題ありません)

### 2. LLMの「学習データ利用」による情報漏洩リスク
生成したソースコードをLLM（ChatGPT, Claude, Geminiなど）のチャット画面に入力する際、プロバイダ側のデータ取扱ポリシーに十分注意してください。

* **学習利用のリスク:** 無料版や一部の標準プランでは、入力したプロンプト（ソースコード）がAIモデルの品質向上（再学習）のために利用される場合があります。これにより、自社の独自コードが他社のサジェスト結果として現れる深刻な情報漏洩リスクが生じます。
* **エンタープライズ保護の推奨:** 業務のコードを扱う際は、**データが学習に利用されないことが明記されているエンタープライズプラン（法人契約）**や、**API経由での利用**を強く推奨します。Web画面を利用する場合でも、設定画面から「モデルのトレーニングにデータを使用しない（オプトアウト）」設定が有効になっていることを必ず確認してください。

### 3. 免責事項
本ツールはユーザーの利便性向上のために提供されるものであり、いかなる保証を伴うものではありません。本ツールを使用したこと、または使用できなかったことにより発生したいかなる直接的・間接的な損害、情報漏洩、その他のトラブル等について、本ツールの開発者および提供者は一切の責任を負いません。ソースコードの抽出・送信および機密情報の取り扱いに際しては、ユーザー自身の責任において十分な確認と安全管理を行ってください。

## 実行モードの詳細
### 1. `all` モード（ディレクトリベース抽出）
指定した対象ディレクトリ（`target_dir`）配下のファイルを走査し、設定ファイルで定義された `includes` と `excludes` の条件に基づいてコードを収集します。プロジェクト全体の構成をLLMに把握させたい場合に適しています。

### 2. `ast` モード（抽象構文木ベースの依存関係抽出・Python専用）
指定したエントリファイル（`entry_file`）内の特定の関数やクラス（`entry_point_name`）を起点とし、Pythonの `ast` モジュールを用いてインポートや参照を解析します。LLMに読ませるコードを「実際に使われている部分」のみに絞り込み、トークン数を大幅に節約したい場合に強力です。
**※標準ライブラリや外部パッケージ（pip等）は自動で除外され、プロジェクト内のファイルのみが抽出されます。**
※ `includes` を併用して、解析から漏れる設定ファイルなどを強制的に追加することも可能です。

## 設定ファイル (JSON) リファレンス
設定はすべてJSONファイルで行います。`configs/` ディレクトリ配下に配置して利用してください。

> [!WARNING]
> **パス指定時の注意点**
> `target_dir` の相対パスは、スクリプトを実行するカレントディレクトリではなく、**設定ファイル（JSON）が置かれているディレクトリを基準**とします。（`config/all/sample.json`の場合、`config/all/`が基準）

### 共通設定・オプション
| キー | 型 | 必須 | 説明 |
| --- | --- | --- | --- |
| `target_dir` | String | **必須** | 解析対象のルートディレクトリ。設定ファイルからの相対パス、または絶対パスで指定します。 |
| `includes` | Array | 任意 | 抽出対象とするファイル/ディレクトリのパスのリスト。`target_dir` からの相対パスで指定します。空配列 `[]` の場合は対象ディレクトリ全体が含まれます。 |
| `excludes` | Array | 任意 | 除外するファイル名やディレクトリ名のパターン（ワイルドカード可）。<br><br>※ `.git`, `__pycache__`, `.venv`, `node_modules`, `*.svg` 等はデフォルトで除外されます。（`core/base_generator.py` にハードコーディングされています） |
| `output_file_name` | String | 任意 | 生成されるMarkdownのファイル名。デフォルトは `context.md` 等。 |
| `enable_tree` | Boolean | 任意 | 出力Markdownの先頭にディレクトリツリーを記載するか（デフォルト: `true`）。 |
| `max_excluded_depth` | Integer | 任意 | ツリーに「除外されたディレクトリ」を表示する際の最大深度（デフォルト: `2`）。 |

### `ast` モード専用設定
| キー | 型 | 必須 | 説明 |
| --- | --- | --- | --- |
| `entry_file` | String | **`ast`のみ必須** | AST解析の起点となるPythonファイルへのパス。**`target_dir` からの相対パスで指定します。** |
| `entry_point_name` | String | **`ast`のみ必須** | 解析の起点となる関数名またはクラス名。 |

## 設定ファイルの例
### 例: `all` モード (`configs/all/sample.json`)
```json
{
  "target_dir": "../../",
  "includes": [],  
  "excludes": [
    "output"
  ],
  "output_file_name": "sample_all_context.md",
  "enable_tree": true,
  "max_excluded_depth": 2
}
```

### 例: `ast` モード (`configs/ast/sample.json`)
```json
{
  "target_dir": "../../",
  "entry_file": "core/generate_ast_snippets.py",
  "entry_point_name": "run",
  "includes": [
    "core/base_generator.py"
  ],
  "excludes": [
    "output"
  ],
  "output_file_name": "sample_ast_context.md",
  "enable_tree": false,
  "max_excluded_depth": 1
}
```

## トラブルシューティング
* **Q. `./run.sh` を実行しようとすると `Permission denied`（許可がありません）エラーが出る**
  * **A.** ターミナルで対象ディレクトリに移動し、以下のコマンドで権限を付与してから再度お試しください。
    ```bash
    chmod +x run.sh
    ```
* **Q. ファイルが多すぎてエラーやフリーズする**
  * **A.** `excludes` に不要なディレクトリ（ログ、ビルドアーティファクト、大規模なデータセットディレクトリなど）を追加して、抽出対象を絞り込んでください。
* **Q. バイナリファイルや画像ファイルはどうなる？**
  * **A.** スクリプトが自動的に非テキストファイルや 100KB を超えるファイルを検知し、スキップします。
* **Q. `ast` モードで一部のPythonファイルが出力されない**
  * **A.** 内部のAST解析はDockerfileで指定されたPython環境（デフォルト: `3.11`）で行われます（ツールが解析できる下限は Python 3.9 になります）。Python 3.12以降の新しい構文が使われているファイルは解析エラー（`SyntaxError`）となりスキップされる場合があります。その際は `core/Dockerfile` の `FROM python:3.11-slim` を `3.12-slim` 等に引き上げるか、`includes` で強制的に対象ファイルを追加してください。
* **Q. 「エラー: JSONのパース失敗、または target_dir が空です。」と表示される**
  * **A.** 現在のスクリプトはシンプルなテキスト検索でJSONを読み取っています。`"target_dir": "..."` の記述が1行に収まるように設定ファイル（JSON）をフォーマットしてください。
